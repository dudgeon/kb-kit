A template repository for context repos

A knowledge base your team and your agents share — plain markdown, versioned in git.

More teams are keeping a knowledge repo: a versioned folder of markdown that people browse and agents maintain. Karpathy's LLM-wiki pattern — an agent that ingests, answers with citations, and lints for rot — is a great starting point for most of them.

kb-kit makes starting one easy. Fork it and the practical parts are already built: markdown rendering on GitHub Pages, basic search, starter skills for setup · ingest · query · lint, and an AGENTS.md so any coding agent knows the conventions.

The LLM-wiki pattern

Karpathy's sketch: knowledge as a wiki that an agent writes and prunes, where every page cites where it came from —

How it works
01

Humans read it

On GitHub, in the site's built-in browser, or in an editor like Obsidian — every page is markdown with relative links. Nothing needs kb-kit to stay readable.

02

Agents work it

AGENTS.md tells any agent the conventions; the skills define the workflows. Ingest files new material into typed pages, query answers with citations, lint checks for rot.

03

Git governs it

Every claim has an author, a date, and a diff. Contributions are pull requests; the audit trail is the version history you already trust for code.

What a fork contains

Starting a fork

GitHub does the hosting — Pages deploys straight from the branch, so it works on locked-down enterprise configs too. The setup skill interviews you, clears the demo content, and writes your kb-card.

$ gh repo fork dudgeon/kb-kit # 1 — your copy
· Settings → Pages → deploy from main # 2 — the whole deployment
$ claude "run the setup skill" # 3 — makes it yours
$ claude "ingest this doc" # 4 — repeat forever

What makes a context repo great?

The kit ships with a ten-dimension taxonomy of KB quality, each graded functional → good → great. It's a checklist while you build, the rubric the lint skill grades against, and the vocabulary of your kb-card.

  1. Scope knows what it's about, and what it's not
  2. Structure front door to any fact in a few hops
  3. Typing like knowledge shaped alike
  4. Linkage relationships made explicit
  5. Provenance every claim traceable to a source
  6. Freshness compounds instead of decaying
  7. Agent-readiness useful to a cold agent in one session
  8. Human experience value without cloning a repo
  9. Discoverability findable and judgeable from outside
  10. Governance clear who decides, with an audit trail
