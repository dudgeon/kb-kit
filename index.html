Why now

Every quarter brings a better model, a new agent framework, a new assistant in your tools. All of it is disposable. The one input that keeps compounding is the thing most teams never write down: context — what you've decided, what you've learned, what "done" means here.

Performance is intelligence × context. The models keep getting smarter on their own. The context is on you.

Agents are disposable

The agent you use today will be replaced in six months. Anything you invest in a single tool's memory is stranded when you switch.

Context is durable

Decisions, definitions, sources, and rationale age slowly. Kept as plain files in git, they outlive every tool that reads them.

Shared beats siloed

An agent's private memory helps one person. A knowledge base in a repo helps every teammate and every agent — the same files, the same truth.

What's a context repo?

A git repository whose product is knowledge. Each page is a markdown file with a little structured metadata (its type, a one-line summary, where the claim came from). An index gives it a front door; a change log gives it a heartbeat; templates keep like knowledge shaped alike.

Humans read it

On GitHub, in this site's built-in browser, or in an editor like Obsidian — it's just markdown with links.

Agents work it

An AGENTS.md schema file tells any agent the conventions, and skills define the workflows: ingest new knowledge, query with citations, lint for rot.

Git governs it

Every claim has an author, a date, and a diff. Contributions are pull requests. The audit trail is the version history you already trust for code.

What's in the kit

Everything a team needs on day one — and nothing that needs a server, a database, or a subscription.

An OKF-based structure

A kb/ bundle following the Open Knowledge Format: typed markdown pages, a progressive-disclosure index, a change log, and grow-as-you-go templates.

Four agent skills

setup (interview-driven fork personalization), ingest, query, and lint — agent-agnostic instructions any assistant can follow.

A kb-card

A model-card for your knowledge base: scope, owner, freshness, and health in one standard file that org crawlers — and curious humans — can find and judge at a glance.

This site

A zero-build GitHub Pages site: this page plus a search-forward KB browser that renders your markdown live. One GitHub Action keeps the search index fresh on every push.

Fork it in five minutes

No build tools, no npm install, no configuration files to learn. GitHub does the hosting; your agent does the setup.

  1. Fork the repo

    Hit Fork on github.com/dudgeon/kb-kit and give it a name your team will recognize.

  2. Turn on Pages

    In your fork: Settings → Pages → Build and deployment → Source: GitHub Actions. That single toggle is the entire deployment setup.

  3. Push (or run the workflow)

    Any push to main — or a manual run of the “Build KB index and deploy to Pages” workflow from the Actions tab — builds the search index and publishes your site at https://<you>.github.io/<repo>/.

  4. Run the setup skill with any agent

    Open the repo with your coding agent (Claude Code, Cursor, Copilot — anything that reads files) and say “run the setup skill”. It interviews you about your team's scope, clears the demo content, and writes your kb-card.

  5. Start ingesting

    Paste a doc, a decision, or a link and say “ingest this.” The agent files it into typed pages, updates the index, and logs the change. Your KB has begun.

What makes a context repo great?

The kit ships with a ten-dimension taxonomy of KB quality — each graded functional → good → great. It's a checklist while you build, a rubric when the lint skill runs, and the vocabulary of your kb-card.

  1. Scope — knows what it's about, and what it's not
  2. Structure — front door to any fact in a few hops
  3. Typing — like knowledge shaped alike
  4. Linkage — relationships made explicit
  5. Provenance — every claim traceable to a source
  6. Freshness — compounds instead of decaying
  7. Agent-readiness — useful to a cold agent in one session
  8. Human experience — value without cloning a repo
  9. Discoverability — findable and judgeable from outside
  10. Governance — clear who decides, with an audit trail
